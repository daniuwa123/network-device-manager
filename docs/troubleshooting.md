## 连接问题

### 症状：无法连接到设备

#### 诊断步骤

**步骤1：检查网络连通性**
```bash
# Windows
ping <device_ip>

# Linux/macOS
ping -c 4 <device_ip>
```

预期：收到设备的回复

如果 ping 失败：
- ❌ 检查IP地址是否正确
- ❌ 检查设备是否开机
- ❌ 检查网络线路是否连接
- ❌ 检查防火墙是否阻止

**步骤2：检查服务端口**

SSH默认端口 22：
```bash
# Linux/macOS
telnet <device_ip> 22

# Windows (需先安装telnet)
netsh int ipv4 set global autojoininterfaceaddresses=enabled
telnet <device_ip> 22
```

预期：看到 SSH banner 或连接成功提示

如果连接失败：
- ❌ SSH服务未启动
- ❌ 防火墙阻止了端口
- ❌ 端口被修改（非标准22端口）

**步骤3：检查用户凭证**

确认：
- ✓ 用户名是否正确
- ✓ 密码是否正确
- ✓ 用户是否有权限

常见默认凭证：
| 厂商 | 用户名 | 密码 |
|-----|-------|------|
| 华为 | admin | admin |
| ZTE | admin | admin |
| 神州数码 | admin | admin |
| 迈普 | admin | admin |

**步骤4：增加连接超时**

如果网络延迟较大：
1. 菜单 → 设置 → 网络
2. 增加连接超时（改为60秒）
3. 重新连接

#### 常见错误信息

| 错误信息 | 原因 | 解决方案 |
|---------|------|--------|
| Connection refused | 端口未开放 | 检查设备SSH/Telnet服务 |
| Connection timeout | 网络无法到达 | 检查网络连通性、防火墙 |
| Authentication failed | 用户名或密码错误 | 重新检查凭证 |
| No route to host | 网络不可达 | 检查路由、网关配置 |
| Permission denied | 用户无权限 | 使用具有更高权限的账户 |

## 命令执行问题

### 症状：命令无法执行或返回错误

#### 问题1：命令语法错误

**症状：**
```
Unknown command "show inteface brief"
```

**解决方案：**
1. 检查命令拼写（interface vs inteface）
2. 查看设备文档确认正确的命令格式
3. 不同厂商命令不同，如：
   - Huawei: `display version`
   - Cisco: `show version`
   - ZTE: `show version`

#### 问题2：权限不足

**症状：**
```
Permission denied
%Error: Access denied
```

**解决方案：**
1. 使用有管理权限的账户登录
2. 进入特权模式（某些厂商需要）：
   ```
   enable
   system-view  # 华为
   configure terminal  # Cisco
   ```

#### 问题3：中文乱码

**症状：**
```
显示中文时出现乱码 ??? 或其他符号
```

**解决方案：**
1. 菜单 → 设置 → 编码
2. 逐个尝试：UTF-8 → GB2312 → GBK
3. 如果还是不行，在命令执行前添加：
   ```
   terminal encoding utf-8
   ```

#### 问题4：命令超时

**症状：**
```
Command execution timeout after 30 seconds
```

**原因：**
- 网络延迟大
- 设备忙碌
- 命令本身执行时间长

**解决方案：**
1. 增加超时时间：
   菜单 → 设置 → 网络 → 命令超时（改为60秒或更长）
2. 检查网络延迟：
   ```bash
   ping -c 100 <device_ip> | grep "min/avg/max"
   ```
   如果平均延迟 > 500ms，考虑优化网络

## 性能问题

### 症状：应用运行缓慢

#### 问题1：内存占用过高

**诊断方法：**
1. 打开系统任务管理器
2. 查看 netman.exe 的内存占用
3. 如果超过500MB，需要优化

**解决方案：**
1. **减少同时连接数**
   - 关闭不使用的连接
   - 一次最多保持5-10个活跃连接

2. **禁用实时监控**
   - 菜单 → 设置 → 监控
   - 禁用自动刷新功能

3. **清理日志**
   - 菜单 → 工具 → 日志管理
   - 清理超过100MB的日志

4. **重启应用**
   - 完全关闭应用
   - 等待5秒后重新启动

#### 问题2：界面卡顿

**症状：** 点击按钮或输入文字响应缓慢

**原因：**
- 设备数量过多（>5000个）
- 正在执行大量网络操作
- CPU占用过高

**解决方案：**
1. **使用分组和搜索**
   - 不要一次加载所有设备
   - 使用搜索过滤需要的设备

2. **启用设备分页**
   - 菜单 → 设置 → 显示 → 分页显示
   - 每页显示100个设备

3. **避免并发操作**
   - 不要同时向多台设备发送大量命令
   - 等待前一个操作完成再进行下一个

#### 问题3：CPU占用过高

**诊断方法：**
1. 打开系统任务管理器
2. 按 CPU 列排序
3. 如果 netman.exe 排名靠前，需要优化

**解决方案：**
1. **停止不必要的后台任务**
   - 禁用自动备份
   - 禁用实时监控
   - 禁用日志滚动显示

2. **减少刷新频率**
   - 菜单 → 设置 → 显示 → 刷新间隔
   - 改为2秒或更长

3. **使用命令行工具处理大型操作**
   - 对于大批量操作，考虑写脚本处理

## 数据问题

### 症状：数据丢失或损坏

#### 问题1：设备列表消失

**原因：**
- 配置文件被删除
- 配置文件损坏

**恢复方案：**

1. **检查备份**
   ```
   C:\Users\<username>\AppData\Roaming\network_tool\backups\
   ```
   - 如果有备份，恢复配置文件

2. **重新导入**
   - 如果有导出的设备列表（CSV/Excel）
   - 菜单 → 数据管理 → 导入设备

3. **联系技术支持**
   - 如果无备份且无导出，可能无法恢复

#### 问题2：配置备份损坏

**症状：** 备份文件无法打开或显示乱码

**解决方案：**
1. 检查备份文件大小
   - 如果大小为0，文件已损坏
   
2. 尝试使用文本编辑器打开
   - 右键 → 打开方式 → 记事本
   
3. 查找以前的备份版本
   - 设备 → 右键 → 配置管理 → 备份历史
   
4. 从设备重新备份
   - 重新连接设备，创建新的备份

## 系统问题

### 症状：应用崩溃

#### 问题1：无法启动应用

**症状：** 双击exe文件无反应

**解决方案：**

1. **检查系统依赖**
   ```bash
   # 检查 Python (如果从源代码运行)
   python --version
   ```

2. **检查权限**
   - 以管理员身份运行
   - 右键 → 以管理员身份运行

3. **清除缓存**
   ```
   删除 C:\Users\<username>\AppData\Local\Temp\netman*
   ```

4. **重新安装**
   - 卸载应用
   - 重启计算机
   - 重新安装应用

#### 问题2：应用在运行中崩溃

**症状：** 应用在某个操作后突然关闭

**诊断步骤：**

1. **查看日志**
   ```
   C:\Users\<username>\AppData\Roaming\network_tool\logs\
   ```
   - 打开 app.log
   - 查看最后的错误信息

2. **检查特定操作**
   - 记住导致崩溃的操作
   - 在GitHub提交Issue时附带此信息

3. **禁用不稳定功能**
   - 临时禁用最近更新的功能
   - 检查是否为某个功能的bug

## 日志分析

### 日志位置

**Windows:**
```
C:\Users\<username>\AppData\Roaming\network_tool\logs\
```

**Linux:**
```
~/.config/network_tool/logs/
```

**macOS:**
```
~/Library/Application Support/network_tool/logs/
```

### 日志文件说明

| 文件名 | 内容 | 用途 |
|-------|-----|------|
| app.log | 应用运行日志 | 诊断应用问题 |
| network.log | 网络通信日志 | 诊断连接问题 |
| audit.log | 操作审计日志 | 记录用户操作 |
| error.log | 错误日志 | 快速定位问题 |

### 日志分析方法

#### 查看错误日志

```bash
# 查看最后20行
tail -f app.log

# 查看包含 "error" 的行
grep -i "error" app.log

# 查看特定时间范围的日志
grep "2024-03-27" app.log
```

#### 常见日志内容解读

**连接错误：**
```
[ERROR] SSH connection failed: Connection timed out
→ 原因：网络不可达或SSH服务未启用
→ 解决：检查网络和SSH服务
```

**认证错误：**
```
[ERROR] Authentication failed: Invalid password
→ 原因：用户名或密码错误
→ 解决：重新检查凭证
```

**命令错误：**
```
[ERROR] Command execution failed: Invalid command syntax
→ 原因：命令语法不正确
→ 解决：检查命令是否适用于该厂商设备
```
