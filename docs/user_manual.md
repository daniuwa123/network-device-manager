## 系统要求

### 硬件要求

| 配置项 | 最低配置 | 推荐配置 |
|-------|--------|--------|
| 处理器 | Intel Pentium 4 | Intel Core i5 |
| 内存 | 512 MB | 2 GB |
| 磁盘 | 200 MB | 1 GB |
| 网络 | 百兆网络 | 千兆网络 |

### 软件要求

| 项目 | 版本 |
|-----|------|
| Windows | 7 SP1 及以上 |
| Linux | Ubuntu 16.04 / CentOS 7 及以上 |
| macOS | 10.12 及以上 |
| Python | 3.8+ (开发环境) |

### 网络要求

- SSH v2 端口：22（或自定义）
- Telnet 端口：23（或自定义）
- HTTP/HTTPS 端口：80/443（可选）
- 网络设备SSH/Telnet服务已启用

## 安装步骤

### Windows 安装

**方式一：使用安装程序（推荐）**

1. 下载 `netman-2.8.6.5-installer.exe`
2. 双击运行，按照向导完成安装
3. 安装完成后自动启动应用
4. 首次运行会初始化配置目录

**方式二：使用便携式版本**

1. 下载 `netman-2.8.6.5-portable.zip`
2. 解压到任意目录
3. 运行 `netman.exe`

### Linux 安装

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.8 python3-pip
pip3 install PyQt6 paramiko requests pyyaml

# 克隆或下载源代码
git clone https://github.com/yourusername/network-device-manager.git
cd network-device-manager
python3 sw.py

# 或创建桌面快捷方式
sudo cp network-device-manager.desktop /usr/share/applications/
```

### macOS 安装

```bash
# 使用Homebrew
brew install python@3.8
pip3 install PyQt6 paramiko requests pyyaml

# 运行应用
python3 sw.py

# 或使用DMG安装程序
# 下载并挂载 netman-2.8.6.5.dmg
# 将应用拖动到 Applications 文件夹
```

## 首次配置

### 启动应用

首次启动时，应用会自动执行以下操作：

1. 创建配置目录
   - Windows: `C:\Users\<username>\AppData\Roaming\network_tool\`
   - Linux: `~/.config/network_tool/`
   - macOS: `~/Library/Application Support/network_tool/`

2. 初始化配置文件
   - `config.json` - 应用配置
   - `device_config.json` - 设备配置
   - `terminal_data.json` - 终端数据

3. 创建日志目录
   - `logs/` - 应用日志

### 管理员账户设置

首次运行时，需要设置管理员账户：

1. 输入用户名（默认：admin）
2. 输入密码（至少8位，建议包含大小写和数字）
3. 确认密码
4. 点击"完成"

> ⚠️ **重要**：记住管理员密码，忘记后无法恢复！

### 初始化配置

1. **网络配置**
   - SSH超时设置：30秒（默认）
   - Telnet超时设置：30秒（默认）
   - 编码格式：UTF-8（默认）

2. **备份设置**
   - 自动备份：禁用（默认）
   - 备份间隔：每天（可修改）
   - 保留备份数：30个（可修改）

3. **显示设置**
   - 界面语言：简体中文（默认）
   - 主题：Light（可选Dark）
   - 字体大小：中等（可调整）

## 基本操作

### 添加设备

#### 方式一：单个添加

1. 右侧面板点击 **"新增设备"** 按钮
2. 填写设备信息：
   - **IP地址**：设备的IPv4或IPv6地址
   - **设备名称**：自定义名称，便于识别
   - **用户名**：登录用户名（默认admin）
   - **密码**：登录密码
   - **厂商**：选择设备厂商或选择"自动检测"

3. 点击 **"保存"** 按钮

#### 方式二：批量导入

1. 点击菜单 → **数据管理** → **导入设备**
2. 选择CSV或Excel文件
3. 确认导入，应用自动解析文件
4. 检查导入结果，点击 **"确认导入"**

**CSV文件格式示例：**
```
IP,Name,Username,Password,Vendor
192.168.1.1,核心交换机,admin,admin123,Huawei
192.168.1.2,接入交换机1,admin,admin123,ZTE
192.168.1.3,接入交换机2,admin,admin123,神州数码
```

### 连接设备

1. 在左侧设备树中选择设备
2. 点击 **"连接"** 按钮，或双击设备
3. 等待连接建立（通常需要3-5秒）
4. 连接成功后，下方会显示设备提示符和可用命令

**连接状态指示：**
- 🟢 绿色：已连接
- 🟡 黄色：连接中
- 🔴 红色：未连接或连接失败

### 执行命令

#### 单条命令

1. 在命令输入框中输入命令
2. 按 **Enter** 或点击 **"执行"** 按钮
3. 等待命令执行完成
4. 在结果区域查看输出

**常见命令示例：**
```bash
# 查看接口状态
display interface brief
show interface status

# 查看VLAN配置
display vlan
show vlan

# 查看系统信息
display version
show version
```

#### 批量命令

1. 在左侧选择多台设备（Ctrl+点击或Shift+点击）
2. 在命令框输入命令
3. 点击 **"批量发送"** 按钮
4. 查看每台设备的执行结果

### 编辑和删除设备

#### 编辑设备

1. 右键点击设备 → **编辑**
2. 修改相关信息
3. 点击 **"保存"**

#### 删除设备

1. 右键点击设备 → **删除**
2. 确认删除
3. 设备被移除

## 高级功能

### 配置备份和恢复

#### 创建备份

1. 连接到设备后，点击 **"备份"** 按钮
2. 系统自动获取并保存配置
3. 备份文件带有时间戳，自动保存到备份目录

#### 查看备份历史

1. 右键点击设备 → **配置管理** → **备份历史**
2. 查看备份列表（时间、大小、状态）
3. 可以对比不同版本的差异

#### 恢复配置

1. 选择要恢复的备份版本
2. 点击 **"恢复"** 按钮
3. 确认操作
4. 系统将配置发送到设备

> ⚠️ **警告**：恢复配置会覆盖现有配置，请谨慎操作！

### VLAN管理

#### 创建VLAN

1. 点击菜单 → **配置管理** → **VLAN管理**
2. 点击 **"新增VLAN"** 按钮
3. 填写信息：
   - VLAN ID：1-4094
   - VLAN名称：便于识别
   - 描述：可选

4. 点击 **"保存"**

#### 配置端口

1. 在VLAN列表中选择VLAN
2. 点击 **"编辑成员"**
3. 选择要加入的端口
   - 选择Access模式：端口属于该VLAN
   - 选择Trunk模式：端口允许该VLAN通过

4. 点击 **"保存"**

### 性能监控

#### 查看实时性能

1. 连接到设备后，点击 **"监控"** 选项卡
2. 可以看到：
   - CPU使用率
   - 内存使用率
   - 接口流量
   - 系统运行时间

3. 右下角可调整刷新间隔

#### 设置告警

1. 点击菜单 → **设置** → **告警**
2. 启用告警功能
3. 设置告警阈值：
   - CPU > 80% 告警
   - 内存 > 85% 告警
   - 接口错误 > 100 告警

4. 保存设置

## 性能优化

### 内存优化

**关闭不必要的功能：**
- 禁用实时监控（如只需要配置管理）
- 关闭过多的连接
- 定期清理日志文件

### 网络优化

**提高连接稳定性：**
- 增加连接超时时间（如网络较慢）
- 启用连接复用（减少重复连接）
- 使用SSH替代Telnet（更安全，传输效率高）

### 大规模设备管理

**管理1000+台设备的建议：**
1. 按地区或业务分组
2. 启用设备分页显示
3. 使用搜索和过滤功能
4. 定期备份配置数据库
5. 在性能较好的机器上运行

### 日志管理

**定期清理日志：**
```bash
# 手动清理（超过100MB）
1. 点击菜单 → 工具 → 日志管理
2. 选择要清理的日志
3. 点击清理
```

## 常见操作流程

### 流程一：新建网络拓扑

1. **创建设备组** → 右侧 → 新建分组
2. **添加设备** → 输入IP和基本信息
3. **验证连接** → 逐个连接测试
4. **生成拓扑** → 菜单 → 工具 → 生成拓扑
5. **导出** → 菜单 → 导出 → 选择格式

### 流程二：标准化部署

1. **创建配置模板** → 菜单 → 配置 → 创建模板
2. **编写配置** → 使用编辑器编写标准配置
3. **保存模板** → 保存为模板库
4. **批量应用** → 选择多台设备 → 应用模板
5. **验证结果** → 备份 → 对比验证

### 流程三：定期备份

1. **启用自动备份** → 设置 → 备份 → 启用自动备份
2. **设置备份策略** → 每天/每周备份
3. **监控备份** → 定期查看备份历史
4. **测试恢复** → 定期测试恢复功能
5. **导出备份** → 定期导出到外部存储
