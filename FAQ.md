# 常见问题解答 (FAQ)

## 📥 安装与启动

### Q1: 如何安装应用？
**A:** 有两种方式：
1. 下载预编译的 `netman-2.8.6.5.exe` 文件并直接运行
2. 从源代码运行：`python sw.py`

### Q2: 安装后打不开怎么办？
**A:** 尝试以下步骤：
1. 确保Windows系统已更新
2. 以管理员身份运行
3. 检查是否被防火墙或杀毒软件拦截
4. 删除 `%APPDATA%/network_tool` 目录并重新启动

### Q3: 提示缺少依赖库怎么办？
**A:** 运行以下命令安装依赖：
```bash
pip install PyQt6 paramiko requests
```

## 🔗 设备连接

### Q4: 为什么连接设备时超时？
**A:** 常见原因和解决方案：
1. **网络问题** - 测试网络连通性：`ping <device_ip>`
2. **设备繁忙** - 等待几秒后重试
3. **端口关闭** - 确认SSH/Telnet服务已启用
4. **防火墙阻止** - 检查防火墙规则
5. **增加超时时间** - 设置 → 网络 → 增加命令超时

### Q5: 支持哪些协议？
**A:** 支持以下协议：
- SSH v2（推荐，安全）
- Telnet（明文传输，不推荐）
- WEB/HTTP（某些设备）

### Q6: 可以连接IPv6设备吗？
**A:** 可以，在IP地址栏输入IPv6地址即可，例如：
```
fe80::1
::1
2001:db8::1
```

### Q7: 支持哪些设备厂商？
**A:** 完整支持列表：
- **华为** (Huawei) - S系列、CE系列
- **中兴** (ZTE) - ZXR系列
- **神州数码** (Dawning) - DCN系列
- **迈普** (Maipu) - MPE系列
- 其他支持Cisco/HP命令集的设备

### Q8: 如何添加新的厂商支持？
**A:** 需要编写厂商驱动程序。详见 [开发文档](docs/development.md#添加新厂商支持)。

## 💻 命令执行

### Q9: 批量发送命令后部分设备失败怎么办？
**A:** 
1. 查看错误日志了解失败原因
2. 逐个检查失败的设备连接
3. 确认命令语法是否正确
4. 检查用户权限是否足够执行该命令

### Q10: 如何执行脚本？
**A:** 有两种方式：
1. 将脚本内容复制到命令框中，逐行执行
2. 使用命令模板：命令 → 保存为模板 → 下次直接应用

### Q11: 命令执行后中文显示乱码怎么办？
**A:** 这是编码问题。解决方案：
1. 设置 → 编码 → 选择 UTF-8
2. 如果问题依然，尝试 GB2312 或 GBK
3. 某些设备可能需要特定编码配置

### Q12: 如何批量修改VLAN配置？
**A:** 步骤：
1. 选择多台设备
2. 右键 → 批量配置 → VLAN管理
3. 设置VLAN参数
4. 点击应用

## 📁 配置备份与恢复

### Q13: 备份文件存储在哪里？
**A:** 备份位置：
- Windows: `C:\Users\<username>\AppData\Roaming\network_tool\backups\`
- Linux: `~/.config/network_tool/backups/`
- macOS: `~/Library/Application Support/network_tool/backups/`

### Q14: 如何恢复配置？
**A:** 
1. 左侧设备树 → 右键 → 配置管理
2. 选择要恢复的备份版本
3. 点击恢复 → 确认

### Q15: 备份文件被删除了怎么办？
**A:** 
1. 检查备份目录是否有 `.bak` 或 `.old` 文件
2. 检查回收站
3. 联系技术支持查看是否有历史备份
4. 从最后一次手动导出的配置文件恢复

### Q16: 如何自动定期备份？
**A:** 
1. 设置 → 备份 → 启用自动备份
2. 设置备份间隔（每天、每周、每月）
3. 设置保留备份数量（默认30个版本）

## 🔍 搜索与过滤

### Q17: 搜索功能不工作怎么办？
**A:** 
1. 确保搜索框中有关键词
2. 检查搜索范围选择是否正确
3. 尝试使用更简单的搜索条件
4. 清除搜索后重试

### Q18: 如何使用正则表达式搜索？
**A:** 
1. 点击搜索框右侧的 `.*` 按钮启用正则
2. 输入正则表达式，例如：`^s.*\d$`
3. 按Enter或点击搜索按钮

## 📊 导入导出

### Q19: 导入CSV文件失败怎么办？
**A:** 检查CSV格式：
```
IP,Name,Username,Password,Vendor
192.168.1.1,Switch1,admin,password,Huawei
192.168.1.2,Switch2,admin,password,ZTE
```

关键要求：
- 第一行必须是表头
- 所有必填列都要有值
- IP地址格式正确
- 编码为UTF-8

### Q20: 可以导出什么格式？
**A:** 支持以下导出格式：
- CSV（逗号分隔）
- Excel（.xlsx）
- JSON（完整配置）

### Q21: 导入大量设备性能如何？
**A:** 
- 一次导入5000个设备通常需要2-3分钟
- 建议分批导入以避免界面卡顿
- 导入过程中会自动跳过重复的IP

## 🔐 权限与安全

### Q22: 如何添加用户账户？
**A:** 
1. 设置 → 用户管理 → 添加用户
2. 输入用户名和密码
3. 设置权限级别
4. 保存

### Q23: 支持哪些权限级别？
**A:** 
- **管理员** - 完全控制
- **操作员** - 可执行命令但不能删除设备
- **查看者** - 只读权限

### Q24: 密码保存在哪里？是否安全？
**A:** 
- 密码使用AES-256加密存储在本地
- 不会上传到网络
- 只有本地用户可以访问
- 建议定期更换管理员密码

### Q25: 如何禁用某些用户？
**A:** 
1. 设置 → 用户管理
2. 选择用户
3. 点击禁用或删除

## 🐛 故障排查

### Q26: 应用崩溃了怎么办？
**A:** 
1. 查看日志文件：`log/app.log`
2. 记下错误信息
3. 在GitHub提交Issue，附加错误日志
4. 尝试清除临时文件后重启

### Q27: 如何查看完整日志？
**A:** 
```
log/
├── app.log           # 应用日志
├── network.log       # 网络日志
├── audit.log         # 操作审计日志
└── error.log         # 错误日志
```

可以用文本编辑器直接打开查看。

### Q28: 性能缓慢，打开很多设备时卡顿？
**A:** 
1. 减少同时打开的设备数
2. 关闭不使用的详细监控
3. 清空过旧的日志文件
4. 增加应用的内存限制

### Q29: 无法保存配置怎么办？
**A:** 检查以下项：
1. 磁盘空间是否充足（至少需要100MB）
2. 是否有写入权限
3. 配置目录是否被占用
4. 尝试手动指定备份位置

## 🌐 网络与远程

### Q30: 可以远程使用吗？
**A:** 
- 本地局域网：支持，推荐
- 跨域远程：支持，需配置VPN或代理
- 云端访问：暂不支持，可使用第三方RDP/VNC

### Q31: 支持代理连接吗？
**A:** 
1. 设置 → 网络 → 代理设置
2. 选择SOCKS5或HTTP代理
3. 输入代理地址和端口
4. 保存

### Q32: 如何配置远程Syslog存储？
**A:** 详见文档 [远程日志存储配置](docs/syslog_storage.md)
